
로그인 정보는 세션에 있다.
세션은 무한히 존재하지 않는다 애플리케이션이 재부팅되거나 timeout 이 지나면 사라진다.
즉, 세션이 사라진다는 것은 로그인 정보 (인증)가 사라진 것이므로 사용자는 다시 세션을 취득하기 위한 행위(로그인)을 해야한다
( 어세스토큰도 결국에는 만료시간을 설정해서 만료하면 재발급을 받든지, 재로그인을 하던지 하는 것처럼 )

세션 타임아웃을 적절한 시간으로 설정하자
시간이 짧으면 UX가 안좋아 질 것이고
시간이 너무 길면 서버 메모리 관리에 있어 비효율적 일것이니 . . (메모리 OOM 리스크 줄이기)
( 요즘엔 토큰 방식을 사용해서 큰 위험으로는 안다가오는 것 같긴하네 . . )
